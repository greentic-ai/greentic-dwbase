## Goal
Implement world lifecycle management so operators and flows can safely manage worlds, and policy atoms have a standard home.

## Deliverables
- Implement `DWBaseEngine::manage_world` for:
  - create world meta
  - archive world
  - resume world
- Standardize policy world convention:
  - `tenant:<id>/policy` (component)
  - `policy:<world>` (node/operator tooling) if needed; document choice
- Add policy atoms (first pass):
  - retention days per world
  - minimum importance threshold per world
  - replication allow/deny per world prefix
- Add CLI support:
  - `dwbase world create|archive|resume`
  - `dwbase world policy set ...` (optional)

## Tests
- Creating a world makes it appear in `worlds()`.
- Archived worlds can be excluded by default.

## Acceptance
- World lifecycle and policy atoms are deterministic and documented.
